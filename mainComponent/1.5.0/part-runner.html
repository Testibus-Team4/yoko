ConfigurePartRunner

We can use part runner as a parameter in the test methods. The runner allows to us to fork work on another thread or JVM (Java Virtual Machine). Below is an example where the JVM = runner.useNewJVMWhenForking(). **

Part runner also allows us to use the bus to communicate with the parts of the program that have been forked. This allows for a lot more detail to be passed from the remote process to the test cases.

All these features of part runner allows you to focus on testing the behaviour which you want to test without dealing with the difficulties of setting up continuous integeation and being tripped up with differneces in developer machine. **
